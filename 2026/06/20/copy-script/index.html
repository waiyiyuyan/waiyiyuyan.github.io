Hexo Landscape美化：仅修改footer.ejs添加代码复制按钮

Hexo 自带的默认主题 landscape 原生没有代码块复制功能，写技术教程贴大量命令、代码片段时，访客只能手动框选复制，体验很差。

这套方案使用纯 CSS+原生 JS，无需安装插件、仅修改单个模板文件，同时兼容开启/关闭代码行号两种页面渲染结构，不会复制出行号数字、不会出现空白复制的问题。

功能特性

  1. 悬浮显示按钮
    默认隐藏复制按钮，鼠标悬浮代码块右上角淡入，不遮挡代码阅读。

  2. 双DOM结构兼容

  • 开启行号：读取 td.code pre，不会复制左侧数字行号
  • 关闭行号：自动读取外层 pre 完整代码

  1. 防重复渲染
    通过 dataset.hasCopy 标记每个代码块，不会重复生成多个复制按钮。

  2. 现代剪贴板API
    弃用废弃的 document.execCommand，使用标准 navigator.clipboard，兼容性更好。

  3. 操作状态提示
    点击按钮切换文字：已复制 / 复制失败 / 无内容，1.2秒自动复原。

  4. 防遮挡层级
    设置 z-index:9999，避免滚动条遮挡复制按钮。

常见问题排查

  1. 悬浮看不到复制按钮
  • 代码粘贴在 footer.ejs 文件最底部
  • 执行 hexo clean 清理缓存后重新生成页面
  • 文章必须使用 ``` 标记代码块，普通文本无高亮容器

  1. 点击提示无内容
    页面DOM渲染异常，重新执行生成命令一般可解决。

  2. 本地直接打开html复制失效
    浏览器安全限制 file:// 本地协议剪贴板权限，部署到 GitHub Pages 等线上服务正常使用。

  3. 按钮被滚动条遮挡
    CSS 已添加 overflow: visible !important，绝大多数场景可解决遮挡。

小结

方案轻量化、零第三方依赖，仅修改 footer.ejs 单个文件全站生效，专门适配 Hexo 默认 Landscape 主题，完美弥补原生缺少代码复制按钮的缺陷，适合追求极简轻量化博客的用户。
