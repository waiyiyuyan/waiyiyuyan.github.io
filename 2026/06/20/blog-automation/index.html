本方案适用于希望构建自动定时爬取资讯发布，同时保留本地高自由度手动撰写随笔，且两端完全不需要同步、绝不产生 Git 冲突的高阶技术博客架构。

📌 一、 核心架构：为什么我们要选择“双端无感解耦”？

在动手建仓库前，必须先看懂这套方案的灵魂——双端解耦，两路贴海报。

  • 传统自动化天坑：爬虫往博客源码库写文章 ➔ 导致云端源码超前 ➔ 本地开机写信时被 Git 拦截拒绝 Push ➔ 逼着你先 git pull ➔ 遇到冲突原地爆炸。
  • 本方案完美闭环：
  • 云端路线：独立爬虫定时起床 ➔ 弹射 daily.md 源码给主仓库 ➔ 主仓库云端 Actions 自动编译 ➔ 将生成的网页强行覆盖推送到 gh-pages 部署分支。
  • 本地路线：你在本地电脑开机自由写你的文章 ➔ 直接在本地运行 hexo g -d ➔ 同样将生成的网页直接推送到 gh-pages 部署分支。

🔑 三、 第二阶段：打通跨仓库跨权限通信的“生死线”

由于爬虫仓库和博客源码仓库是独立的，爬虫要想跨仓库把文件送过去，必须有一把主仓库颁发的特制高级钥匙（Personal Access Token）。

Step 1：申请 GitHub 全局高级密钥（PAT）

  1. 鼠标点击 GitHub 右上角头像 ➔ 选择 Settings。
  2. 左侧菜单拉到最下方，点击 Developer settings。
  3. 选择 Personal access tokens ➔ 点击 **Tokens (classic)**。
  4. 点击右侧 Generate new token ➔ 选择 **Generate new token (classic)**。
  5. Note 随便填（例如：crawler-to-blog-key），Expiration（有效期）建议选 No expiration（永不过期）。
  6. 关键权限勾选：将 repo 这一大项前面的复选框全部勾选（包含其子项 repo:status、repo_deployment 等）。
  7. 滚动到页面最下方，点击 Generate token。
  8. ⚠️ 必须立刻复制：页面上会弹出一串以 ghp_ 开头的长字符串。立刻把它复制到你的本地记事本中，只要刷新或者关闭这个网页，你就再也看不到它了！

🕷️ 四、 第三阶段：独立爬虫仓库（ygpy-crawler）的搭建

在 GitHub 上创建一个全新、完全独立的仓库，命名为 **ygpy-crawler**。

Step 1：把高级钥匙注入爬虫仓库

  1. 打开新创建的 ygpy-crawler 仓库网页端 ➔ 点击 ⚙️ Settings。
  2. 左侧菜单找到 Secrets and variables ➔ 展开并点击 Actions。
  3. 点击右侧的大绿按钮 New repository secret。
  4. Name 必须严格填写：BLOG_DEPLOY_TOKEN。
  5. Secret 框内粘贴你刚才在记事本里保存的、以 ghp_ 开头的特制高级密钥。点击 Add 按钮保存。

Step 2：编写 Node.js 爬虫脚本（crawl.js）

在本地为一个新文件夹进行 npm init -y，安装 npm install axios cheerio dayjs。创建 crawl.js 脚本。

🏗️ 五、 第四阶段：主博客源码库（waiyiyuyan.github.io）的云端配置

当爬虫把文章弹射过来时，主博客源码库必须有一个“接收并自动化打包”的机制。

Step 1：解除云端 Actions 的“403 权限死锁”

⚠️ 避坑提醒（编译完报错 403 变白）：GitHub 出于系统安全考虑，默认情况下任何仓库自带的内部 Actions 只有读权限。当它编译完想往你的 gh-pages 分支写网页时，会被系统当场拦截并报 403 错误，导致你的博客全盘崩溃变白。

  1. 打开你的博客主源码仓库（waiyiyuyan.github.io）网页端 ➔ 点击 ⚙️ Settings。
  2. 左侧菜单找到 Actions ➔ 点击其子项 General。
  3. 页面一路向下拉，找到 Workflow permissions（工作流权限）区域。
  4. 将默认选项改选为：Read and write permissions（读和写权限）。
  5. 点击 Save 按钮保存。

🏁 六、 终极胜利：属于你的“双端并行”无感日常工作流

到这一步，整条大动脉已经彻底调顺打通，所有的报错死结被解开。你在未来的日子里，可以享受到如下令人极度舒适的自动化体验：

🤖 1. 云端自动化闭环（完全不打扰你）

每天早上 08:30，爬虫仓库触发 run.yml ➔ 运行 crawl.js ➔ 自动把当天最新的 Markdown 文章塞进主仓库的 main 分支 ➔ 主仓库检测到 main 变动，自动运行 deploy.yml ➔ 19秒 完成完美编译，23秒 自动覆盖发布到 gh-pages 分支。

你想写你自己的随笔了，完全不需要管云端今天有没有更新过文章，直接在本地电脑进行操作：

  2. 写完之后，在本地终端直接敲下通关口诀：
