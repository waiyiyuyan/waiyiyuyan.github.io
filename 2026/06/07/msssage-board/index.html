用 cloudflare 搭建留言板源码

正在进行重大更新，暂时不要部署这个代码了 2026-06-10，
代码马上就写好了，感谢朋友们的留言，我在写帖子的时候，代码写的很随意。后来发现有人真的给我留言时，我才重视起来。
帖子里教的那个版本，使用的是轮询查询数据库，每隔几秒就查一次，这样做非常简单，但是数据库的压力巨大，
一天才500万次的读写额度，它能读200多万次。10条帖子，轮询一次就要读10次，1000条，一次就读1000次，这根本不能用。
我目前在做的这个版本，采用websocket的方式，只在首次加载页面时，读取一次数据库，然后有新信息的时候，只读写一条数据，页面只渲染一次有变化的帖子。性能大大提升。
最终版是使用分页机制(就差这一步我就完成了)读取数据库，哪怕有一万条数据，一次只读取几条，使用动态读取数据库的方式，这样整个程序大概能达到商业化的水平了。
使用免费的就是这样，限制太多。这不行那不行的，要不然也不至于改来改去。不过很快就好了，也就这几天就写好了 2026-06-12

留言系统 D1 数据表设计文档（Cloudflare D1 / SQLite）

一、数据表总览

整套留言系统共包含 3 张核心数据表，分别存储主帖子、楼中楼回复、用户@通知，表之间通过 ID 关联联动。

表名 作用
message 存储一级主帖子/主留言，是整个留言系统的主体数据
reply 存储主帖对应的楼中楼回复，每条回复均关联一条主帖
notify 存储用户被@的消息通知，包含已读状态、消息来源、内容预览等

二、分表详细说明

1. 主帖子表 message

用途：存放用户发布的一级主留言、帖子，发帖、列表查询、帖子删除等业务均操作此表。

字段明细

字段名 数据类型 约束 字段含义 业务逻辑说明
id INTEGER 主键、自增 帖子唯一标识ID 作为关联回复、通知的外键关联ID
name TEXT 非空 发帖人昵称 展示发帖用户名称，支持昵称查重校验
content TEXT 非空 帖子正文内容 用户发布的文字留言主体
create_time DATETIME 默认当前时间 发帖时间 用于列表排序、前端时间展示
media_urls TEXT 可空，默认空字符串 图片/媒体链接集合 存储帖子内上传的图片URL，多图可拼接字符串存储
uid TEXT 可空 用户唯一标识 区分不同登录/匿名用户，用于权限、数据归属判断

建表语句 + 索引（适配 Cloudflare D1）

2. 回复表 reply

用途：存放所有针对主帖的楼中楼回复，每条回复强绑定对应主帖，支持附带图片资源。

字段明细

字段名 数据类型 约束 字段含义 业务逻辑说明
id INTEGER 主键、自增 回复唯一标识ID 通知表关联该ID，精准定位对应回复
msg_id INTEGER 非空 关联主帖ID 对应 message.id，标识回复归属的主帖子
r_name TEXT 非空 回复人昵称 展示回复者名称，参与昵称查重校验
r_content TEXT 非空 回复正文内容 用户输入的楼中楼回复纯文本（后端不再拼接 回复XX： 前缀）
create_time DATETIME 默认当前时间 回复时间 前端展示、回复列表排序使用
media_urls TEXT 可空，默认空字符串 图片/媒体链接集合 存储回复内附带的图片URL
uid TEXT 可空 用户唯一标识 回复用户身份标识，用于数据归属识别
to_user TEXT 可空，默认空字符串 被回复人昵称 存储本条回复的目标对象昵称，前端渲染「XX回复了你/回复了XX」

建表语句 + 索引（适配 Cloudflare D1）

3. 消息通知表 notify

用途：记录留言中@用户产生的提醒消息，支持已读/未读状态标记，为消息提醒功能提供数据源。

字段明细

字段名 数据类型 约束 字段含义 业务逻辑说明
id INTEGER 主键、自增 通知唯一标识ID 标记已读、删除通知时的操作主键
target_msg_id INTEGER 非空 关联主帖ID 跳转通知时定位到对应的主帖子
receive_uid TEXT 非空 被@用户标识 接收通知的目标用户，通知列表按此字段筛选
is_read INTEGER 默认 0 消息已读状态 0=未读（小红点提醒），1=已读（无提醒）
create_time TIMESTAMP 默认当前时间 通知生成时间 前端展示通知创建时间、排序使用
reply_id INTEGER 可空 关联回复ID 精准定位到触发@操作的单条回复
reply_name TEXT 可空 发起@的用户昵称 前端展示「XX@了你」等文案
post_title TEXT 可空 主帖标题 通知列表展示帖子标题，区分消息来源
reply_preview TEXT 可空 回复内容预览 截取部分回复文本，用于通知弹窗/列表简略展示

建表语句 + 索引（适配 Cloudflare D1）

  1. 数据库环境：基于 Cloudflare D1（底层为 SQLite），语法完全通用，可直接部署使用；
