用 cloudflare 搭建留言板源码

本次分享一套全功能在线留言板解决方案，技术栈采用 Cloudflare Workers + D1 SQLite，无需本地服务器，依托 Cloudflare 边缘网络即可快速上线。

程序自带随机昵称生成、帖子发布、多层回复、@好友通知、消息红点提醒、回复折叠、帖子删除等交互功能。整套程序共依赖三张数据表，也是项目最关键的部分。本文重点对每张数据表的用途、字段、约束、索引进行详细说明，同时提供一键建表 SQL 语句，前端 JS 源码可直接复制使用，帮助大家快速搭建、维护这款留言板。

一、数据表总览

整套程序共使用 3 张数据表，分工明确，缺一不可：

表名 作用
message 存储主帖子 / 一级留言（用户点击悬浮按钮发布的内容）
reply 存储所有楼中楼回复（针对主帖的评论）
notify 存储 @用户 消息通知，记录被@人、消息来源、已读状态

二、分表详细说明

1. 主帖子表 message

用途：存放用户发布的一级主留言/帖子，发帖、查询列表、删除帖子均会操作此表。

字段明细

字段名 数据类型 约束 字段含义 代码对应逻辑
id INTEGER 主键、自增、非空 帖子唯一 ID 删帖、关联回复时作为关联 ID
title TEXT 非空 帖子标题 前端表单提交的 title 字段
name TEXT 非空 发帖人昵称 昵称查重接口会查询该字段
content TEXT 非空 帖子正文内容 前端输入的留言主体
create_time TEXT 非空、默认当前时间 发帖时间 列表排序、前端展示时间

建表语句 + 索引

2. 回复表 reply

用途：存放所有针对主帖的楼中楼回复，每条回复都会绑定一条主帖。

字段明细

字段名 数据类型 约束 字段含义 代码对应逻辑
id INTEGER 主键、自增、非空 回复唯一 ID 通知表会关联此 ID
msg_id INTEGER 非空 关联主帖 ID 对应 message.id，标识回复归属的主帖
r_name TEXT 非空 回复人昵称 昵称查重接口会查询该字段
r_content TEXT 非空 回复内容 用户输入的回复文本（包含 @昵称）
create_time TEXT 非空、默认当前时间 回复时间 前端展示、列表排序

建表语句 + 索引

3. 消息通知表 notify

用途：存储 @昵称 产生的提醒消息，支持标记已读功能，为消息铃铛提供数据源。

字段明细

notify 表字段说明（Markdown 完整版，对齐线上实际结构）

字段名 数据类型 约束 字段含义 代码对应逻辑
id INTEGER 主键、自增 通知唯一 ID 标记已读接口根据此 ID 更新数据
target_msg_id INTEGER 非空 关联主帖 ID 跳转消息时定位到对应主帖
receive_uid TEXT 非空 被 @的用户昵称 接收通知的用户，消息列表按此字段筛选
is_read INTEGER 默认 0 已读状态 0 = 未读（小红点），1 = 已读（灰色文字）
create_time TIMESTAMP 默认当前时间 通知生成时间 前端展示通知时间

post_title TEXT 无 主帖标题 通知列表展示对应帖子标题，区分消息来源
